## US-006: Implement Multi-Speaker Synthesis in translate.py - COMPLETE

**Date:** 2026-01-15 23:55
**Files changed:**
- translate.py (added synthesize_segments_multi_speaker function)

**Implementation notes:**
- Created new function synthesize_segments_multi_speaker() at line 496
- Speaker matching algorithm uses timestamp overlap:
  - For each transcription segment, find diarization segment with maximum overlap
  - Calculate overlap as: max(0, min(seg_end, diar_end) - max(seg_start, diar_start))
  - Return speaker with highest overlap, fallback to first speaker if no overlap
- Uses speaker-specific voice samples from speaker_samples dict
- Fallback mechanism: if speaker not in speaker_samples, use first available sample
- Preserves natural 1x speed playback from US-005 (no time-stretching)
- Function signature: (segments, diarization_segments, speaker_samples, lang_code, total_duration, tracker, update_fn) -> Path

**Key design decisions:**
- Overlap-based matching is more robust than point-in-time matching
- Handles edge cases where transcription and diarization boundaries don't align perfectly
- Progress display shows which speaker is being synthesized for each segment

**Learnings:**
- Diarization segments and transcription segments have different boundaries
- Overlap calculation is the best way to match them
- The existing synthesize_segments() is kept for backward compatibility (single-speaker mode)


## US-007: Add Background Audio Mixing - COMPLETE

**Date:** 2026-01-15 (iteration 7)
**Files changed:**
- translate.py (added mix_audio_with_background function)

**Implementation notes:**
- Created new function mix_audio_with_background() at line 653
- Uses ffmpeg filter_complex for professional audio mixing:
  - Speech track at 100% volume (first input)
  - Background track at configurable volume (default 30%)
  - apad filter pads shorter background with silence
  - amix with duration=first ensures output matches speech duration
- Function signature: (speech_path, background_path, output_path=None, background_volume=0.3) -> Path
- Volume parameter clamped to 0.0-1.0 range for safety

**Key ffmpeg filter:**
```
[1:a]apad,volume={volume}[bg];[0:a][bg]amix=inputs=2:duration=first:dropout_transition=0
```
- apad: Pad background with silence if shorter than speech
- volume: Apply configurable volume to background
- amix: Mix two audio streams, output duration=first (speech)
- dropout_transition=0: No fade at end

**Learnings:**
- ffmpeg amix filter is the cleanest way to mix audio streams with volume control
- Using duration=first ensures output matches the speech track length
- apad is essential to handle backgrounds shorter than speech

```

-metadata:s:s:0 language=eng  # Set subtitle language metadata
```

```python

```

- ffmpeg uses ISO 639-2 (3-letter) codes for language metadata, but our app uses ISO 639-1 (2-letter)
- The -metadata:s:s:0 option sets metadata for stream 0 of the subtitle track
- Backward compatibility maintained - existing callers don't need to change
